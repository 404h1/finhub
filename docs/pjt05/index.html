Django 기본 인증 모델을 커스텀 유저로 확장하고, 게시판 권한 체계를 로그인 기반으로 완성합니다.

F501
커스텀 유저 모델 구현
Django 기본 User 모델을 AbstractUser로 확장합니다. nickname, interest_stocks(쉼표 구분), profile_image 필드를 추가하여 사용자 정보를 풍부하게 저장합니다.

심화
FinHub 구현

FinHub에서
어떻게 적용했나

accounts 앱에 커스텀 유저 모델을 도입하고, 투자 성향 분석까지 연결한 완전한 사용자 시스템을 구축했습니다.

1

AbstractUser 기반 커스텀 모델

AUTH_USER_MODEL = 'accounts.User'로 설정하고 nickname, interest_stocks, profile_image 필드를 추가. makemigrations 전에 반드시 settings에 등록합니다.

2

UserCreationForm 커스터마이징

기본 UserCreationForm을 상속해 닉네임·관심종목·프로필 이미지 필드를 추가. 회원가입 성공 시 login()으로 자동 로그인 처리합니다.

3

@login_required + 작성자 검증

게시글 작성 뷰에 @login_required 데코레이터를 적용. 수정·삭제 뷰에서 post.author != request.user.username 조건으로 권한을 검증합니다.

4

프로필 통합 조회 페이지

프로필 페이지에서 유저 정보와 Post.objects.filter(author=user.username)로 내 게시글 목록을 통합 제공합니다.

5

LLM 투자 성향 분석 (심화)

사용자의 최근 게시글 텍스트를 수집해 OpenAI Chat API로 전송. "공격형 / 중립형 / 안정형" 등 투자 성향 레이블과 이유를 자연어로 반환합니다.
